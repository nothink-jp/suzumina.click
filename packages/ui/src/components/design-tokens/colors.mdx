すべての色の正準ソースは `src/styles/globals.css`（`:root` / `.dark`）です。本ドキュメントのスウォッチは
`hsl(var(--token))` で実際の CSS 変数を参照しているため、globals.css を変更すればここも自動的に追従します
（手書きの 16 進スナップショットは持ちません）。

実装では **semantic トークン（`primary` など）を優先**し、`suzuka-*` / `minase-*` の生スケールは bespoke な
ブランド表現にのみ使用します（README の idiom 準拠）。すべて WCAG 2.1 AA を想定した設計です。

## Heart Accent（差し色・桜霞）

瞳・ハートの鮮やかピンクレッド。くすませた `suzuka` に対する差し色として、**お気に入り / いいね の active 状態・新着強調**専用に使う。
status（info/success/…）と同じ **semantic role トークン**として消費する（`bg-heart` / `text-heart`、面上の文字は `text-heart-foreground`、
hover は `/90`）。数字付きの生スケール（`heart-500` 等）は持たない＝消費層は常に semantic。
**淡背景 `bg-heart/10` の上に `text-heart` を載せない**（実測コントラスト 4.08 で AA 不合格・axe 検出済み。SPR-254。
強調が要る場面は solid の `bg-heart text-heart-foreground` を使う）。

## Dark Mode

祖先要素に `.dark` クラスを付与するとトークンが切り替わります。semantic トークンは値が差し替わり、
`suzuka-*` / `minase-*` の生スケールは **明暗が反転**します（`suzuka-50 ↔ 950`、`100 ↔ 900` …）。
値はすべて `globals.css` の `.dark` スコープが正準です。

### Semantic（light → dark, HSL）

| トークン                | Light        | Dark         |
| ----------------------- | ------------ | ------------ |
| `--background`          | 340 40% 99%  | 326 16% 7%   |
| `--foreground`          | 326 16% 13%  | 340 30% 96%  |
| `--card`                | 0 0% 100%    | 326 14% 11%  |
| `--primary`             | 340 58% 46%  | 340 58% 64%  |
| `--secondary`           | 30 38% 66%   | 28 24% 24%   |
| `--muted-foreground`    | 324 8% 40%   | 330 8% 64%   |
| `--accent`              | 330 24% 95%  | 326 12% 16%  |
| `--destructive`         | 0 72% 45%    | 0 62.8% 30.6% |
| `--border` / `--input`  | 330 20% 91%  | 326 10% 16%  |
| `--ring`                | 340 58% 46%  | 340 58% 64%  |

### 生スケールの反転（例）

| スケール段   | Light        | Dark         |
| ------------ | ------------ | ------------ |
| `suzuka-50`  | 342 70% 97%  | 336 48% 14%  |
| `suzuka-500` | 340 58% 46%  | 340 58% 55%  |
| `suzuka-950` | 336 48% 14%  | 342 70% 97%  |
| `minase-50`  | 36 50% 97%   | 25 28% 18%   |
| `minase-950` | 25 28% 18%   | 36 50% 97%   |

（端の段ほど明暗がそのまま入れ替わり、中央の `500` は明度のみ微調整される。）

## 使用ガイドライン

実装は **semantic トークン優先**。生スケールは bespoke なブランド表現にのみ使う。

| ロール          | 推奨クラス（semantic 優先）                       | 用途                          |
| --------------- | ------------------------------------------------- | ----------------------------- |
| Primary Action  | `bg-primary text-primary-foreground`              | メイン CTA、フォーム送信       |
| Secondary       | `bg-secondary text-secondary-foreground`          | 補助アクション                 |
| Muted           | `bg-muted text-muted-foreground`                  | 抑制面・ヒントテキスト         |
| Accent (hover)  | `bg-accent`                                       | ホバー/アクティブ面            |
| Destructive     | `bg-destructive text-destructive-foreground`      | 削除・エラー                   |
| Heart (accent)  | `bg-heart text-heart-foreground` / `text-heart`   | お気に入り/いいね active・新着   |
| Border          | `border-border`                                   | 既定の境界線                   |
| Focus Ring      | `ring-ring`                                        | フォーカス表示                 |
| Brand (bespoke) | `bg-suzuka-500` / `bg-suzuka-50 border-suzuka-200` | ブランド色の面・カード         |
| Brand Text      | `text-suzuka-800` / `text-suzuka-900`             | ブランド見出し                 |

## ブランド役割と生スケールの規律（[Brand roles]）

- **suzuka（くすみローズ）= primary = 主役**。主要操作・アクティブ状態・リンク・ブランド面の大半はこちら。
- **minase（ミルクティー）= secondary = 補助・限定**。特別機能の識別・強調の差し色に限り、多用しない（footer 等の構造的な面は semantic に寄せる）。
- **heart（鮮やかピンクレッド）= お気に入り/いいねの差し色**。くすませた suzuka に対する pop。status と同じ **semantic role**（`bg-heart`/`text-heart`）で消費し、対象は favorite/like の active と新着強調に限定。

生スケール（`suzuka-*` / `minase-*` の直書き）を使うときの規律（apps 実装の判断基準）:

1. **薄背景・本文・リンク・枠は semantic に寄せる** — `bg-muted` / `bg-card`、`text-foreground` /
   `text-muted-foreground`、リンクは `text-primary`、枠は `border-border`。これらは `.dark` 値を持つので
   ダーク対応が自動で付く。
2. **生スケールは「意図的なブランド演出（hero / グラデ CTA / ブランドタグ等）」だけに使い、必ず `dark:` を併記する**。
   生スケールはダークで**明暗が反転**する（`suzuka-50 ↔ 950`）ため、`dark:` 無しの `bg-suzuka-50` はダークで濃い背景になり意図が壊れる。
   例: `bg-suzuka-50` → `bg-suzuka-50 dark:bg-suzuka-950`、`text-minase-800` → `text-minase-800 dark:text-minase-200`。
3. **minase（ミルクティー）面の文字は暗色か `minase-50` 等の極淡色に限定**。白文字は AA 不適合（白 / `minase-500` = 2.2:1）。
4. **two-tone グラデ（`from-suzuka-500 to-minase-500` 等）は CTA 等の意図的な場面のみ**。ページ背景の地として安易に使わない。

> 例: footer やセクション背景のような**構造的な面は semantic**（footer = `bg-muted` + `border-t border-border`）に
> 寄せてテーマに自動追従させる。生スケール＋`dark:` は hero やグラデ CTA のような**意図的なブランド演出だけ**に限る。

## アクセシビリティ

- **Primary**: `bg-primary`（= suzuka-500 / `340 58% 46%`・くすみローズ）+ `text-primary-foreground`（≒白）で **白文字 ≈5.7:1** ✓（彩度 75%→58% に下げつつ SPR-129 の AA を維持）
- **本文 + 背景**: `bg-suzuka-50` + `text-suzuka-900`（高コントラスト）✓
- **Secondary（ミルクティー）**: 明色のため **暗色文字**（`text-secondary-foreground` ≈7.4:1）で AA を取る。白文字は不適合
- **Heart**: `bg-heart` + `text-heart-foreground`（≒白）≈4.9:1 ✓。`text-heart` は白上 ≈4.9:1 ✓。**`bg-heart/10` 上は実測 4.08 で AA 不合格**（axe 検出・SPR-254）— 淡色地に heart 文字を載せない
- 隣接した明度（`suzuka-200` + `suzuka-400` など）の組み合わせは避ける
