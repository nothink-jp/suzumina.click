## 🏗️ システム構成

### カスタマイゼーション分類

| 重要度 | 対象 | 対応方法 |
|--------|------|----------|
| **Critical** | button.tsx, dialog.tsx | 自動テスト必須 |
| **Medium** | 10コンポーネント | 選択的更新 |
| **Low** | 35コンポーネント | 手動確認 |

### 更新プロセス

1. **Phase 1: 更新前確認**（1時間）
   - 環境準備・バックアップ
   - カスタマイゼーション状態確認
   - 利用可能更新の調査

2. **Phase 2: 更新実行**（1時間）
   - Critical コンポーネント更新
   - Medium コンポーネント選択的更新
   - data-slot属性再適用

3. **Phase 3: テスト・検証**（30分）
   - 自動テスト実行
   - 手動動作確認
   - 完了チェック
